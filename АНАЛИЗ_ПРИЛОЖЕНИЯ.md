## ❌ Критические проблемы

### 1. **Тестовый код в продакшене**
**Файл**: `SettingsView.swift` (строки 100-132)
- Кнопки генерации тестовых данных и очистки всех тренировок доступны пользователям
- **Риск**: Пользователь может случайно удалить все данные
- **Решение**: Удалить секцию "🧪 TESTING" или добавить флаг `#if DEBUG`

### 2. **Отсутствие обработки ошибок для пользователя**
**Файлы**: `DataManager.swift`, `WorkoutViewModel.swift`
- Ошибки сохраняются только в консоль (`print`)
- Пользователь не видит, когда что-то пошло не так
- **Решение**: Добавить алерты/тосты для критических ошибок

### 3. **Нет валидации входных данных**
**Проблема**: Пользователь может ввести отрицательные значения веса/повторов
- Нет проверки на разумные пределы (например, вес > 500 кг)
- **Решение**: Добавить валидацию в `ConfigureExerciseView.swift` и другие формы

### 4. **Отсутствие резервного копирования**
- Нет iCloud синхронизации
- Нет экспорта данных (CSV, JSON)
- При потере устройства все данные теряются
- **Решение**: Добавить iCloud Drive или экспорт в файлы

### 5. **Магические числа в коде**
**Файл**: `BodyHeatmapView.swift` (строки 96-103)
```swift
let femaleFrontOffset: CGFloat = 25   // Почему 25?
let femaleBackOffset: CGFloat = -10  // Почему -10?
```
- **Решение**: Вынести в константы с комментариями или конфигурацию

## ⚠️ Проблемы и улучшения

### 6. **Дублирование данных в модели Exercise**
**Файл**: `Workout.swift` (строки 77-82, 214-218)
- Старые поля (`sets`, `reps`, `weight`) хранятся для совместимости
- Это может привести к рассинхронизации данных
- **Решение**: После миграции всех данных удалить старые поля или добавить валидацию

### 7. **Нет подтверждения при удалении**
**Файл**: `SettingsView.swift` (строка 116)
- Кнопка "Clear All Workouts" удаляет данные без подтверждения
- **Решение**: Добавить `.alert()` с подтверждением

### 8. **Отсутствие индикаторов загрузки**
- Нет loading states для асинхронных операций (расчет восстановления, загрузка данных)
- **Решение**: Добавить `ProgressView` или skeleton screens

### 9. **Хардкод значений восстановления**
**Файл**: `NotificationManager.swift` (строка 86)
```swift
let recoveryHours = savedHours > 0 ? savedHours : 48.0  // Хардкод 48 часов
```
- **Решение**: Сделать настраиваемым в Settings

### 10. **Много print() в продакшене**
- 35+ вызовов `print()` в коде
- **Решение**: Использовать систему логирования (OSLog) или убрать для релиза

### 11. **Нет обработки поврежденных данных**
**Файл**: `DataManager.swift`
- Если JSON файл поврежден, приложение просто возвращает пустой массив
- Пользователь не узнает о проблеме
- **Решение**: Добавить проверку целостности и восстановление из бэкапа

### 12. **Отсутствие undo/redo**
- Удаление тренировок необратимо
- **Решение**: Добавить корзину или undo функционал
